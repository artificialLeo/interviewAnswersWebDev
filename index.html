DOM

The DOM is a cross-platform and language-independent interface that treats an HTML document as a tree structure wherein each node is an object representing a part of the document. The DOM represents a document with a logical tree.

DOM

DOM – это объектная модель документа, которую браузер создаёт в памяти компьютера на основании HTML-кода, полученного им от сервера. DOM – это набор связанных объектов, созданных браузером при парсинге её текста.

Doctype

DOCument TYPE — A code in an HTML page that informs the Web browser which version of the markup language was used to write the page in order to render it properly on screen. Typically the first line of code in the page, the DOCTYPE points to a Document Type Definition (DTD), which contains the markup rules.

Doctype

DOCTYPE — это определение типа документа. Благодаря этой записи, браузер определяет, какая в данном документе используется версия (Document Type Definition). Нужен он для того чтобы браузеры правильно отображали разметку документа. Если не указать DOCTYPE, то браузер будет добавлять "что то от себя", а это убивает кроссбраузерность.

SQL

SQL

SQL — язык программирования структурированных запросов, который используется для оптимизации и обслуживания баз данных, поиска их частей, обновления, извлечения из базы и удаления.

TypeScript

TypeScript (TS) is a superset of JavaScript (JS), among the main features of which are the possibility of explicit static type assignment, support for classes and interfaces. One of the serious advantages of TS over JS is the ability to create, in various IDEs, such a development environment that allows you to identify common errors right in the process of entering the code. The use of TypeScript in large projects can lead to increased reliability of programs, which, at the same time, can be deployed in the same environments where regular JS applications run.

TypeScript

TypeScript (TS) – это надмножество JavaScript (JS), среди основных особенностей которого можно отметить возможность явного статического назначения типов, поддержку классов и интерфейсов. Одним из серьёзных преимуществ TS перед JS является возможность создания, в различных IDE, такой среды разработки, которая позволяет, прямо в процессе ввода кода, выявлять распространённые ошибки. Применение TypeScript в больших проектах может вести к повышению надёжности программ, которые, при этом, можно разворачивать в тех же средах, где работают обычные JS-приложения.

MongoDb

MongoDb is a document-oriented database, unlike traditional relational databases such as MySQL or PostgreSQL, it does not use a tabular representation with foreign key relationships, based on the principle of storing documents in Binary JSON format. Those each record is a document, without a hard-coded schema, that can contain sub documents.

MongoDb

MongoDb - это документо-ориентированная база данных, в отличие от традиционных реляционных баз данных, таких как MySQL или PostgreSQL не использует табличный способ представления со связями через внешние ключи, основанная на принципе хранении документов в Binary JSON формате. Т.е. каждая запись это документ, без жестко заданной схемы, который может содержать вложенные документы.

GraphQL

GraphQL is a syntax that describes how to query data, and is mainly used by the client to download data from the server. GraphQL has three main characteristics:
1) Allows the client to specify exactly what data he needs.
2) Facilitates the aggregation of data from multiple sources.
3) Uses a type system to describe data.

GraphQL

GraphQL это синтаксис, который описывает как запрашивать данные, и, в основном, используется клиентом для загрузки данных с сервера. GraphQL имеет три основные характеристики:
1) Позволяет клиенту точно указать, какие данные ему нужны.
2 )Облегчает агрегацию данных из нескольких источников.
3) Использует систему типов для описания данных.

OOP

OOP is a programming methodology based on representing a software product as a set of objects, each of which is an instance of a specific class. OOP uses the interaction of objects as basic elements.
Main principles:
Abstraction, inheritance, encapsulation, polymorphism.

ООП

ООП - методология программирования, основанная на представлении программного продукта в виде совокупности объектов, каждый из которых является экземпляром конкретного класса. ООП использует в качестве базовых элементов взаимодействие объектов.
Главные принципы:
Абстрацкия, наследование, инкапсуляция, полиморфизм.

Java Access Modifiers

private: (used in constructors, inner classes, methods and class fields) - Access is only allowed in the current class.
default (package-private): (used in classes, constructors, interfaces, inner classes, methods and class fields) - Package level access. If the class is declared like this, it will only be available inside the package.
protected: (used in constructors, inner classes, methods, and class fields) Access modifier at the package level and in the inheritance hierarchy.
public: (used in classes, constructors, interfaces, inner classes, methods and class fields) - The access modifier is public, available to everyone.

Java модификаторы доступа

private: (используется в конструкторах, внутренних классах, методах и полях класса) - Доступ разрешен только в текущем классе.
default (package-private): (используется в классах, конструкторах, интерфейсах, внутренних классах, методах и полях класса) - Доступ на уровне пакета. Если класс будет так объявлен он будет доступен только внутри пакета.
protected: (используется в конструкторах, внутренних классах, методах и полях класса) Модификатор доступа на уровне пакета и в иерархии наследования.
public: (используется в классах, конструкторах, интерфейсах, внутренних классах, методах и полях класса) - Модификатор доступа общественный, доступен всем.

Java Collection Framework

Java Collection Framework is a hierarchy of interfaces and their implementations that is part of the JDK and allows the developer to use a large number of data structures "out of the box".
Collection - This interface is part of the JDK since version 1.2 and defines the basic methods for working with simple collections of elements that will be common to all implementations.
The Map interface is also included in the JDK since version 1.2 and provides the developer with basic methods for working with key-value data.
Collection methods were implemented directly in the interface as default methods.

Java Collection Framework

Интерфейс Map также в составе JDK c версии 1.2 и предоставляет разработчику базовые методы для работы с данными вида «ключ — значение».

Java Generics

Generics are parameterized types that add type safety to the language.
Parameterized types allow you to declare classes, interfaces, and methods, where the data type they operate on is specified as a parameter. Using generics, you can create a single class, for example, that will automatically work with different data types.
Classes, interfaces, or methods that deal with parameterized types are called parameterized or generics.

Java Дженерики

Дженерики (или обобщения) - это параметризованные типы которые добавляют в язык безопасность типов.
Параметризованные типы позволяют объявлять классы, интерфейсы и методы, где тип данных, которыми они оперируют, указан в виде параметра. Используя дженерики, можно создать единственный класс, например, который будет автоматически работать с разными типами данных.
Классы, интерфейсы или методы, имеющие дело с параметризованными типами, называются параметризованными или обобщениями.

JS Prototype inheritance

In JavaScript, each object can have an association with another object - a so-called "prototype". If the search for a certain property (or method) in the original object fails, the interpreter tries to find the property of the same name (method) in its prototype, then in the prototype of its prototype, etc. For example, if we requested a call to object .property , JavaScript starts looking for this in the object itself, then in the object prototype, then in a prototype of the prototype, and so on until the end.

JS Прототипное наследование

В JavaScript каждый объект может иметь ассоциацию с другим объектом - так называемый "прототип". В случае, если поиск некоторого свойства (или метода) в исходном объекте заканчивается неудачно, интерпретатор пытается найти одноименное свойство (метод) в его прототипе, затем - в прототипе прототипа и т. д. К примеру, если мы затребовали обращение к object .property, JavaScript начинает искать свойство property в самом объекте object, затем – в прототипе object, прототипе прототипа object, и так до конца.

JS Closure

In simple words, this is an internal function, i.e. function in function.
A closure is called a closure because after the parent or outer function has been executed, the inner code still lives in the interpreter and can be executed. Errors will not occur even if the inner function (closure) uses variables from the outer function.
function makeAdder(x) {
  return function(y) {
   return x + y;
  };
}
var add5 = makeAdder(5);
var add10 = makeAdder(10);
console.log(add5(2)); // 7
console.log(add10(2)); // 12

JS Замыкание

Простыми словами это внутренняя функция, т.е. функция в функции.
Замыканием называют потому, что после выполнения родительской или внешней функции, код внутренней все еще "живет" в интерпретаторе, и его можно выполнить. Ошибок не возникнет даже если внутренняя функция (замыкание) будет использовать переменные из внешней функции.
function makeAdder(x) {

}
var add5 = makeAdder(5);

JS Promise

A Promise is an object that represents the final completion (or failure) of an asynchronous operation and its resulting value.
A promise has four states:
Pending is the initial state of the promise. The result of the promise is unknown because the operation has not completed.
Fulfilled - The asynchronous operation is complete and has a result.
Rejected - Asynchronous operation failed, cause exists.
Completed - Completed or Rejected.

The promise constructor takes resolve and reject as parameters. Resolve records the result of the operation, and reject records the reason for the failure of the operation. The result can be handled in the .then method, an error in the .catch method. The .then method also returns a promise, so we can use a chain of multiple .then.

JS Promise

Promise — это объект который представляет окончательное завершение (или сбой) асинхронной операции и ее результирующее значение.
У промиса есть четыре состояния:
Ожидание — начальное состояние промиса. Результат промиса неизвестен, поскольку операция не завершена.
Выполнено — асинхронная операция выполнена, имеется результат.
Отклонено — асинхронная операция не выполнена, имеется причина.
Завершено — выполнено или отклонено.

В качестве параметров конструктор промиса принимает resolve и reject. В resolve записывается результат выполнения операции, в reject — причина невыполнения операции. Результат может быть обработан в методе .then, ошибка — в методе .catch. Метод .then также возвращает промис, поэтому мы можем использовать цепочку, состоящую из нескольких .then.

URI

URI (Uniform Resource Identifier) is a sequence of characters that identifies a physical or abstract resource that does not have to be accessible via the Internet, and the type of resource that will be accessed is determined by the context and / or mechanism.

URI

URI (унифицированный идентификатор ресурса) - последовательность символов, идентифицирующая физический или абстрактный ресурс, который не обязательно должен быть доступен через сеть Интернет, причем, тип ресурса, к которому будет получен доступ, определяется контекстом и/или механизмом.

URL

URL (Uniform Resource Locator) - a site location indicator on the Internet, identifies the resource and determines the location of the resource with the way to access it. The URL serves as a standardized way to record the address of a resource on the Internet; the URL contains the domain name and an indication of the path to the page, including the file name of that page.

URL

URL ( определитель местонахождения ресурса ) —указатель размещения сайта в интернете, идентифицирует ресурс и определяет местонахождение ресурса со способом обращения к нему. URL служит стандартизированным способом записи адреса ресурса в сети Интернет, URL-адрес содержит доменное имя и указание пути к странице, включая название файла этой страницы.

URN

URN (Uniform Resource Name) is the unique name of the object. The URN includes the name of the namespace and the identifier in that space. It only defines the name of a specific item, which can be found in many specific locations.

URN

URN (унифицированное имя ресурса) - является уникальным именем объекта. URN включает в себя название пространства имен и идентификатора в этом пространстве. Он определяет только название конкретного предмета, который может находится во множестве конкретных мест.

Internet Protocol

The Internet Protocol is a network layer protocol of the Open Systems Interconnection network model and refers to protocols that organize connections based on circuit switching. It is a low-level routing network protocol that breaks data into small packets and sends them through the network to a specific address, which does not guarantee delivery of all these packets to that address.

Интернет Протокол

Интернет протокол - протокол сетевого уровня сетевой модели Open Systems Interconnection и относится к протоколам, которые организуют соединения на основе коммутации каналов. Является низкоуровневым маршрутизирующим сетевым протоколом, разбивающим данные на небольшие пакеты и посылающим их через сеть по определенному адресу, что не гарантирует доставки всех этих пакетов по этому адресу.

TCP

TCP (Transmission Control Protocol) is a higher layer network protocol that allows packets to be bound, sorted and retransmitted to ensure reliable data delivery.

TCP

ТСР (Протокол Управления Передачей) - сетевой протокол более высокого уровня, обеспечивающим связывание, сортировку и повторную передачу пакетов, чтобы обеспечить надежную доставку данных.
